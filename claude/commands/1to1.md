L'utilisateur veut préparer son 1/1 avec un managee. L'argument fourni est : $ARGUMENTS

## Configuration

Lis le fichier `~/.claude/team.json` pour charger l'arborescence complète de l'équipe (noms, Slack IDs, emails, GitHub handles, Notion IDs, etc.).

Identifie le managee à partir de l'argument (match partiel OK, insensible à la casse, check aussi le champ "aliases"). Si l'argument ne correspond à aucun managee, liste les noms disponibles et demande à l'utilisateur de préciser.

## Collecte de données

Utilise des agents en parallèle pour collecter toutes les données. La période couverte est les 7 derniers jours (calculer la date de début dynamiquement).

### 1. Page 1:1 Notion

Utilise `mcp__claude_ai_Notion__notion-fetch` avec le champ `notion_1to1_page` du managee. Lis les dernières entrées datées pour identifier :
- Les sujets abordés au dernier 1:1
- Les actions en cours ou en attente
- Les points ouverts non résolus

### 2. Slack — Messages envoyés par le managee

Utilise `mcp__claude_ai_Slack__slack_search_public_and_private` avec :
- query = `from:<@{slack_id}> after:YYYY-MM-DD`
- sort = "timestamp", sort_dir = "desc", response_format = "concise"

Note les sujets sur lesquels le managee s'est exprimé, les discussions importantes, les éventuels blocages ou frustrations exprimés.

Pour les messages qui semblent importants (décisions, questions, problèmes), lis le thread complet avec `mcp__claude_ai_Slack__slack_read_thread`.

### 3. Gmail — Emails échangés

Utilise `mcp__claude_ai_Gmail__gmail_search_messages` avec :
- q = `from:{email} OR to:{email} after:YYYY/MM/DD`
- maxResults = 30

Lis les emails importants (pas les notifications automatiques). Note les sujets d'échange.

### 4. Board Produit-Tech — Cartes du managee

Utilise `mcp__claude_ai_Notion__notion-search` avec :
- query = le prénom du managee
- data_source_url = le champ `board_data_source` du team.json

Cherche les cartes qui sont assignées au managee (champ Assign ou Squad) et qui ont bougé récemment (Last edited time dans les 7 derniers jours). Note :
- Les cartes en cours (Dev in progress, QA, etc.)
- Les cartes terminées (Done, In prod)
- Les cartes bloquées
- Les changements de statut récents

### 5. GitHub — PRs mergées

Utilise Bash pour lister les PRs mergées par le managee sur le repo (champ `repo` du team.json) :

```bash
gh pr list --repo {repo} --author {github} --state merged --search "merged:>YYYY-MM-DD" --limit 20 --json title,url,mergedAt,additions,deletions
```

Note le volume d'activité et les sujets des PRs.

### 6. Équipe (si le managee a role=manager et une team non vide)

Pour CHAQUE membre de l'équipe du managee (champ `team` dans team.json), collecte en parallèle :

a) **Slack** — `from:<@{slack_id}> after:YYYY-MM-DD` (response_format = "concise", include_context = false)
   - Sujets sur lesquels la personne s'est exprimée
   - Blocages ou frustrations éventuels

b) **GitHub** — PRs mergées :
   ```bash
   gh pr list --repo {repo} --author {github} --state merged --search "merged:>YYYY-MM-DD" --limit 10 --json title,url,mergedAt,additions,deletions
   ```

c) **Board Produit-Tech** — Cartes assignées ayant bougé récemment (même méthode que pour le managee)

PAS besoin de chercher les emails ni la page 1:1 Notion pour les membres d'équipe.

## Résumé

Une fois TOUTES les données collectées, rédige un briefing en texte simple (pas de markdown riche).

```
BRIEFING 1:1 — {Prénom} — {date du jour}
==========================================

DEPUIS LE DERNIER 1:1
---------------------
- Points ouverts du dernier 1:1 et leur avancement
- Actions qui étaient assignées et leur statut

SA SEMAINE
----------
- Ce sur quoi il/elle a travaillé (PRs, cartes board, sujets Slack)
- Volume d'activité (nb PRs, nb cartes bougées)
- Sujets de discussion majeurs sur Slack

POINTS D'ATTENTION
------------------
- Blocages ou frustrations détectés
- Cartes bloquées ou en retard
- Sujets de tension ou désaccords
- Choses qui pourraient nécessiter ton aide

EMAILS / ÉCHANGES DIRECTS
--------------------------
- Résumé des échanges email importants

ÉQUIPE
------
Pour chaque membre de l'équipe, un mini-résumé :

  {Prénom} :
  - Activité : {nb PRs mergées}, {cartes en cours}
  - Sujets notables : {résumé 1-2 lignes}
  - Attention : {blocages, frustrations, inactivité inhabituelle}

Vue d'ensemble équipe :
- Vélocité globale (volume PRs, cartes terminées)
- Patterns : qui est bloqué, qui avance bien, déséquilibre de charge
- Sujets transverses ou récurrents

SUJETS À ABORDER
-----------------
- Suggestions de sujets à aborder basées sur l'analyse ci-dessus
- Points équipe à remonter au managee
```

N'inclure que les sections qui ont du contenu pertinent. Être concis mais complet. La section ÉQUIPE n'apparaît que si le managee a des reports directs.
